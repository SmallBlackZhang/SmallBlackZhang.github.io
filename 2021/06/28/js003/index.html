数据类型转换

javaScript是一种弱类型语言，变量不受类型限制，所以在特定情况下我们需要对类型进行转换。
「类型转换」分为显式类型转换和隐式类型转换。每种转换又分为原始类型转换和对象类型转换。

显式类型转换

显式类型转换就是我们所说强制类型转换

ToBoolean 转换

除了特殊的几个值 “”、 undefined、 NaN、 null、 false、 0 转换为 Boolean 为 false 之外，其他类型值都转换为 true。

ToString 转换

数据类型 String 类型
String 不转换
Number 转换为数字对应的字符串
Bigint 转换为数字对应的字符串
Boolean 转换为字符串 “true”和”false”
null 转换为字符串 “null”
undefined 转换为字符串 “undefined”
Symbol 转换为 Symbol 对应的字符串
Object 执行 ToPrimitive(input,'String')转换

ToNumber 转换

数据类型 数字类型
Number 不转换
String 字符串中只有数字转换为对应的数字,空串转换为 0，其他转换为 NaN
Boolean true 转换为 1 false 转换为 0
null 0
undefined NaN
Bigint 不建议转换
Symbol 报错 TypeError: Cannot convert a Symbol value to a number
Object 先执行 ToPrimitive(input,'Number')转换为基本数据类型,再执行 ToNumber转换为 Number 类型

ToPrimitive 转换

引用类型在转原始类型的时候，会根据ToPrimitive 算法规则转换。
//以简单的 function 做例子来描述 ToPrimitive 算法

function ToPrimitive(input, type) {}

  • 如果 input 为基本数据类型 直接返回 input 本身
  • 如果 input 为引用数据类型
    • 如果 type 有值，执行下一步
    • 如果 type 没有传值
      • input 为 Date 类型 type = String
      • 其他类型 type = Number
  • 根据 type 类型执行 toString 和 valueOf
    • type 为 string
      • 调用 input 对象的 toString()方法，如果值是原值则返回
      • 否则，调用 input 对象的 valueOf()方法，如果值是原值则返回
      • 否则，抛出 TypeError 错误
    • type 为 number
      • 调用 input 对象的 valueOf()方法，如果值是原值则返回
      • 否则，调用 input 对象的 toString()方法，如果值是原值则返回
      • 否则，抛出 TypeError 错误

拓展：内置的 valueOf()和 toString()是可以进行重写的

想完全理解上面的题目，你还需要对隐式类型转换需要一定的了解

隐式类型转换

隐式类型转换不需要人为的强制类型转换，javaScript 自动将类型转换为需要的类型，所以称之为隐式类型转换.

怎样会发生隐式类型转换?
因为javaScript并没有类型声明，所以任意两个变量或字面量，都可以进行运算，不同类型之间的数据运算会将数据变为相同类型然后进行运算，这就产生了隐式类型转换。

算数运算符相关类型转换

  • 加法(+)
  • 减法(-)
  • 乘法(*)
  • 除法(/)
  • 取余(%)
  • 一元正值(+)
  • 一元负值(-)
  • 自增(++)
  • 自减(–)
特殊的加法运算符

js 的加法(+)运算符既可以计算两个数的和也能实现字符串的拼接，当两个不同类型值通过加法(+)运算符运算符连接时又会产生各种各样的隐式类型转换
那么任意两个变量或字面量通过加法(+)运算符连接时，怎么进行隐式类型转换呢

  • a+b
  • 两边同时执行 ToPrimitive转换 即 ToPrimitive(a)+ToPrimitive(b)
  • 判断 ToPrimitive(a)和 ToPrimitive(b)得到的结果有没有 String 类型
    • 有 String 类型
      • 两个数再执行 ToString() 即 ToString(ToPrimitive(a))+ToString(ToPrimitive(b))
      • 进行拼接 得到拼接的字符串结果
    • 没有 String 类型
      • 两个数再执行 ToNumber() 即 ToNumber(ToPrimitive(a))+ToNumber(ToPrimitive(b))
      • 进行加法运算 得到的数字结果

举几个栗子

  • “number” + 15
    • 两边同时 ToPrimitive 转换，因为都是基本数据类型 => “number” + 15
    • 有 String 类型 两边再执行 ToString 转换 => “number” + “15”
    • 拼接字符串 => “number15”
  • true + false
    • 两边同时 ToPrimitive 转换 => true + false
    • 两边再执行 ToNumber 转换 => 1 + 0
    • 进行加法运算 => 1
  • new Date() + true
    • 两边同时 ToPrimitive 转换 => “Wed Jun 30 2021 11:30:58 GMT+0800 (新加坡标准时间)” + true
    • 两边再执行 ToString 转换 => “Wed Jun 30 2021 11:30:58 GMT+0800 (新加坡标准时间)” + “true”
    • 拼接字符串 => “Wed Jun 30 2021 11:30:58 GMT+0800 (新加坡标准时间)true”
  • []+{}
    • 两边同时 ToPrimitive 转换 => “” + “[object Object]”
    • 两边再执行 ToString 转换 => “” + “[object Object]”
    • 拼接字符串 => “[object Object]”
其他算数运算符

对运算符相关变量或字面量，执行ToNumber(ToPrimitive(input,'Number')),然后进行算数运算

举几个栗子

  • “number” - 15
    • 两边同时 ToNumber(ToPrimitive(input,’Number’))转换 => NaN - 15
    • 进行减法运算 => NaN
  • true * false
    • 两边同时 ToNumber(ToPrimitive(input,’Number’))转换 => 1 * 0
    • 进行运算 => 0
  • new Date() / false
    • 两边同时 ToNumber(ToPrimitive(input,’Number’))转换 => 1625024813205 / 0
    • 进行运算 => Infinity
  • +{}
    • ToNumber(ToPrimitive(input,’Number’)) => + NaN
    • NaN

  • 等于(==)
  • 不等于(!=)
  • 全等(===)
  • 不全等(!==)
  • 大于(>)
  • 小于(<)
  • 大于等于(>=)
  • 小于等于(<=)
全等(===)和不全等(!==)

全等(===)和不全等(!==)运算符两边的数据不会进行隐式数据类型转化

  • 1、如果类型不同，就不相等
  • 2、如果两个都是数值，并且值相等，那么[相等]；!例外的是，如果其中至少一个是 NaN，那么不相等NaN与任何数值比较结果都是false。
  • 3、如果两个都是字符串，每个位置的字符都一样，那么相等；否则不相等 。
  • 4、如果两个值都是 true，或者都是 false，那么相等。
  • 5、如果两个值都引用同一个对象或函数，那么相等；否则不相等。
  • 6、如果两个值都是 null，或者都是 undefined，那么相等。

举几个栗子

  • true === 1 // false
  • true === true / true
  • +0 === -0 // true
  • NaN === NaN // false
  • {}==={} // false
  • null === null // true
  • null === undefined // false
等于(==)和不等于(!=)
  • 如果类型相同
    • 不进行隐式类型转化，执行全等于比较
  • 如果类型不同
    • 先对引用类型数据执行 ToPrimitive，再判断类型是否相同
    • 如果类型相同，执行全等于比较
    • 类型不同，同时对两个数据执行 ToNumber 转换,然后进行数字大小比较

举几个栗子

  • “true” == true
    • NaN == 1
    • false
  • [“x”] == “x”
    • “x” == “x”
    • true
  • [] == ![]
    • [] == false
    • “” == false
    • 0 == 0
    • true
  • [undefined] == false
    • “” == false
    • 0 == 0
    • true
其他比较运算符
  • 对引用类型数据进行ToPrimitive(input,'Number')转换
  • 如果两边都是 String 类型
    • javascript字符串在进行大小比较时，会根据第一个不同的的 ASCII 码值进行比较
    • 如果这一位相同，取下一位字符的 ASCII 码值进行比较
  • 否则，执行ToNumber转换,然后进行数字比较

举几个栗子

  • [“1”] > false
    • 1 > 0
    • true
  • new Date() > “1609430400000” （2021-01-01 时间戳）
    • 1625031825645 > 1609430400000
    • true
  • ‘2021-2’ < ‘2021-11’
    • 前 5 位字符（2021-）的 ASCII 值一直相同
    • 对第 6 位字符的 ASCII 值比较 ‘2’<’1’
    • false

逻辑运算符相关类型转换

  • 逻辑与 (&&)
  • 逻辑或 (||)
  • 逻辑非 (!)

对运算符相关变量或字面量，执行ToBoolean

举几个栗子

  • if( [undefined] && [null] )
    • if(true && true)
    • if(true)
  • if( 0 || “0” && {} )
    • if((false || true) && {})
    • if(true && true)
    • if(true)
  • !{}
    • !true
    • false
